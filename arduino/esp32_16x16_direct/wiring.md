# 16x16 点阵屏 接线确认指南

## ⚠️ 重要：先确认你的点阵屏类型

**16 针的 16x16 点阵屏** 比较少见，请先确认：

### 方法 1：看背面芯片

拍个照片看看背面有没有芯片：

| 芯片型号 | 驱动类型 | 需要 GPIO |
|---------|---------|----------|
| MAX7219 | 串行驱动 | 3 个 (DATA, CLK, CS) |
| 74HC595 | 移位寄存器 | 3 个 (DATA, CLK, LATCH) |
| 无芯片 | 直接驱动 | 32 个 (16 行 +16 列) |
| HT1632C | 专用驱动 | 3-4 个 (DATA, CLK, CS, WR) |

### 方法 2：数引脚功能

用万用表或看丝印：

```
如果 16 针是：
- 8 行 + 8 列 → 实际是 8x8 点阵
- 16 行 + 公共端 → 需要 16 个 GPIO
- DATA/CLK/CS + 电源 → 有驱动芯片
```

## 常见情况分析

### 情况 A：背面有 MAX7219 芯片
→ 用第一个版本代码 (`esp32_16x16_dotmatrix/`)

### 情况 B：背面有 74HC595 芯片
→ 用第二个版本代码 (`esp32_16x16_74hc595/`)

### 情况 C：背面没芯片，纯 16 针
→ 可能是 **8x8 点阵屏**，不是 16x16！

8x8 点阵屏典型接法：
```
16 针 = 8 行 + 8 列

ESP32 需要 16 个 GPIO：
行：GPIO 25,26,27,14,12,13,15,2
列：GPIO 32,33,35,34,39,36,4,16
```

## 临时测试代码

如果不确定，先用这个测试：

```cpp
// 测试哪个引脚能点亮 LED
void setup() {
  Serial.begin(115200);
  // 把所有 GPIO 设为输出
  for(int i = 0; i < 40; i++) {
    pinMode(i, OUTPUT);
  }
}

void loop() {
  // 依次点亮每个引脚
  for(int i = 0; i < 40; i++) {
    digitalWrite(i, HIGH);
    Serial.println("Testing pin: " + String(i));
    delay(500);
    digitalWrite(i, LOW);
  }
}
```

上传后观察哪个引脚能让点阵屏亮，就能确定接线了。
